# <a name="using-grunt-in-aspnet-core"></a>С помощью Grunt в ASP.NET Core 

По [Ноэл Риса](https://blog.falafel.com/falafel-software-recognized-sitefinity-website-year/)

Grunt-это средство запуска задач JavaScript, который автоматизирует минификации сценария, компиляцию TypeScript, средства «корпус» качество кода, предварительной процессоров CSS и практически любой повторяющихся тяжелой задачей, должен образом для поддержки разработки клиентских. Grunt полностью поддерживается в Visual Studio на то, что шаблоны проектов ASP.NET по умолчанию используется Gulp (см. [с помощью Gulp](using-gulp.md)).

В этом примере использует пустой проект ASP.NET Core начальной точкой, чтобы показать, как автоматизировать процесс построения клиента с нуля.

Готовый пример удаляет целевой каталог развертывания, объединяет файлы JavaScript, проверяет качество кода, также сжимает содержимое файла JavaScript и развертывает в корневой каталог веб-приложения. Мы будем использовать следующие пакеты:

* **grunt**: средство запуска задач Grunt пакет.

* **Очистка grunt contrib**: подключаемый модуль, который удаляет файлы или каталоги.

* **grunt-contrib-jshint**: подключаемый модуль, который просматривает качества кода JavaScript.

* **grunt-contrib-concat**: подключаемый модуль, который объединяет файлы в один файл.

* **grunt contrib uglify**: подключаемый модуль, который уменьшает JavaScript для уменьшения размера.

* **grunt contrib Контрольные значения**: подключаемый модуль, который отслеживает действия файла.

## <a name="preparing-the-application"></a>Подготовка приложения

Чтобы начать, задать новый пустой веб-приложения и добавьте файлы пример TypeScript. Файлы typeScript автоматически компилируется в JavaScript с использованием параметров по умолчанию Visual Studio и будут нашей сырье для обработки с помощью Grunt.

1.  В Visual Studio создайте новый `ASP.NET Web Application`.

2.  В **новый проект ASP.NET** диалоговое окно, выберите ASP.NET Core **пустой** шаблона и нажмите кнопку "ОК".

3.  Проверьте структура проекта в обозревателе решений. `\src` Папка содержит пустой `wwwroot` и `Dependencies` узлов.

    ![пустой веб-решений](using-grunt/_static/grunt-solution-explorer.png)

4.  Добавить новую папку с именем `TypeScript` в каталог проекта.

5.  Перед добавлением все файлы, убедитесь, что Visual Studio имеет параметр "компиляции при сохранении" для возвращенных файлов TypeScript. *Сервис > Параметры > текстовый редактор > Typescript > проекта*

    ![Параметры автоматического compliation файлов TypeScript](using-grunt/_static/typescript-options.png)

6.  Щелкните правой кнопкой мыши `TypeScript` каталог и выберите **Добавить > новый элемент** в контекстном меню. Выберите **файл JavaScript** элемента и присвойте файлу имя *Tastes.ts* (Примечание \*.ts расширения). Скопируйте строку кода TypeScript под в файле (при сохранении, новый *Tastes.js* появится файл с исходного кода JavaScript).
    
    ```typescript
    enum Tastes { Sweet, Sour, Salty, Bitter }
    ```

7.  Добавьте второй файл, чтобы **TypeScript** каталог и назовите его `Food.ts`. Скопируйте приведенный ниже код в файл.

    ```typescript
    class Food {
      constructor(name: string, calories: number) {
        this._name = name;
        this._calories = calories;
      }
    
      private _name: string;
      get Name() {
        return this._name;
      }
    
      private _calories: number;
      get Calories() {
        return this._calories;
      }
    
      private _taste: Tastes;
      get Taste(): Tastes { return this._taste }
      set Taste(value: Tastes) {
        this._taste = value;
      }
    }
    ```

## <a name="configuring-npm"></a>Настройка NPM

Настройте NPM для загрузки grunt и grunt задачи.

1. В обозревателе решений щелкните правой кнопкой мыши проект и выберите **Добавить > новый элемент** в контекстном меню. Выберите **файл конфигурации NPM** товара, оставьте имя по умолчанию *package.json*и нажмите кнопку **добавить** кнопки.

2. В *package.json* файла внутри `devDependencies` объекта фигурные скобки, введите «grunt». Выберите `grunt` из Intellisense списка и нажмите клавишу ВВОД. Visual Studio Квота grunt имя пакета и добавьте двоеточие. Справа от запятой выберите последняя стабильная версия пакета в верхней части списка Intellisense (нажмите клавишу `Ctrl-Space` Если Intellisense не отображается).

    ![grun Intellisense](using-grunt/_static/devdependencies-grunt.png)
    
    > [!NOTE]
    > Использует NPM [семантического управления версиями](http://semver.org/) организовать зависимостей. Семантического управления версиями, также известный как SemVer определяет пакеты с формирование <major>.<minor>. <patch>. IntelliSense упрощает семантического управления версиями, отображая только несколько распространенных вариантов. Верхний элемент в списке Intellisense (0.4.5 в приведенном выше примере) считается последняя стабильная версия пакета. Знак вставки (^) соответствует самой последней основной номер версии, а тильда (~) соответствует последнему дополнительному номеру версии. В разделе [NPM semver версия средства синтаксического анализа ссылки](https://www.npmjs.com/package/semver) в соответствии с полной функциональность, предоставляющий SemVer.

3. Добавьте дополнительные зависимости, для загрузки grunt-contrib -\* пакетов для *чистой*, *jshint*, *concat*, *uglify*и *Контрольные значения* как показано в следующем примере. Версии не требуется в соответствии с примером.

    ```json
    "devDependencies": {
      "grunt": "0.4.5",
      "grunt-contrib-clean": "0.6.0",
      "grunt-contrib-jshint": "0.11.0",
      "grunt-contrib-concat": "0.5.1",
      "grunt-contrib-uglify": "0.8.0",
      "grunt-contrib-watch": "0.6.1"
    }
    ```

4. Сохранить *package.json* файла.

Пакеты для каждого элемента devDependencies будут загружены и все файлы, необходимые для каждого пакета. Можно найти файлы пакета в `node_modules` каталога, позволяя **Показать все файлы** кнопку в обозревателе решений.

![grunt node_modules](using-grunt/_static/node-modules.png)

> [!NOTE]
> Если необходимо, можно восстановить вручную, зависимости в обозревателе решений щелкните правой кнопкой мыши на `Dependencies\NPM` и выбрав **восстановить пакеты** пункт меню.

![Восстановление пакетов](using-grunt/_static/restore-packages.png)

## <a name="configuring-grunt"></a>Настройка Grunt

Grunt настраивается с помощью манифест с именем *Gruntfile.js* , определяет, загружает и регистрирует задачи, которые можно запускать вручную или с запуском автоматически на основе событий в Visual Studio.

1.  Щелкните правой кнопкой мыши проект и выберите **Добавить > новый элемент**. Выберите **файла конфигурации Grunt** , то оставьте имя по умолчанию *Gruntfile.js*и нажмите кнопку **добавить** кнопки.

    Исходный код включает определение модуля и `grunt.initConfig()` метод. `initConfig()` Используется для задания параметров для каждого пакета и остальная часть модуль будет загрузить и зарегистрировать задачи.
    
    ```javascript
    module.exports = function (grunt) {
      grunt.initConfig({
      });
    };
    ```

2. Внутри `initConfig()` метод, добавьте параметры для `clean` задач, как показано в примере *Gruntfile.js* ниже. Задачу "Очистить" принимает массив строк каталогов. Эта задача удаляет файлы из wwwroot/lib и удаляет весь/временный каталог.

    ```javascript
    module.exports = function (grunt) {
      grunt.initConfig({
        clean: ["wwwroot/lib/*", "temp/"],
      });
    };
    ```

3. Под методом initConfig(), добавьте вызов `grunt.loadNpmTasks()`. Это сделает задачи запускаемых из Visual Studio.

    ```javascript
    grunt.loadNpmTasks("grunt-contrib-clean");
    ```

4. Сохранить *Gruntfile.js*. Файл должен выглядеть примерно на снимке экрана ниже.

    ![Начальное gruntfile](using-grunt/_static/gruntfile-js-initial.png)

5. Щелкните правой кнопкой мыши *Gruntfile.js* и выберите **Task Runner Explorer** в контекстном меню. Откроется окно Task Runner Explorer.

    ![меню обозревателя средство запуска задач](using-grunt/_static/task-runner-explorer-menu.png)

6. Убедитесь, что `clean` области отображается **задачи** в Task Runner Explorer.

    ![Список задач задач runner explorer](using-grunt/_static/task-runner-explorer-tasks.png)

7. Щелкните правой кнопкой мыши задачу "Очистить" и выберите **запуска** в контекстном меню. Окно командной строки отображает ход выполнения задачи.

    ![Задача runner explorer запустите задачу "Очистить"](using-grunt/_static/task-runner-explorer-run-clean.png)
    
    > [!NOTE]
    > Отсутствуют файлы или каталоги для очистки еще. При желании можно создать их вручную в обозревателе решений и запустите задачу "Очистить" в качестве теста.
    
8. В этом методе initConfig(), добавьте запись для `concat` ниже коде.

    `src` Массива свойства перечислены файлы для объединения, в том порядке, они должны быть объединены. `dest` Свойство путь присваивается объединенного файла, который создается.

    ```javascript
    concat: {
      all: {
        src: ['TypeScript/Tastes.js', 'TypeScript/Food.js'],
        dest: 'temp/combined.js'
      }
    },
    ```
    
    > [!NOTE]
    > `all` Свойство в приведенном выше коде это имя целевого объекта. Целевые объекты используются в некоторых задачах Grunt для разрешения нескольких средах построения. Позволяет просмотреть встроенных конечных объектов, с помощью Intellisense или назначить собственные.
    
9. Добавить `jshint` задачи, используя приведенный ниже код.

    Программа jshint качества кода выполняется для каждого файла JavaScript, найденные в каталоге temp.
    
    ```javascript
    jshint: {
      files: ['temp/*.js'],
      options: {
        '-W069': false,
      }
    },
    ```

    > [!NOTE]
    > Параметр «-W069» ошибка произведенное jshint при JavaScript используется квадратная скобка синтаксис, т. е. присвоение свойства вместо точечной нотации `Tastes["Sweet"]` вместо `Tastes.Sweet`. Этот параметр отключает предупреждения, чтобы разрешить оставшаяся часть процедуры, чтобы продолжить.

10.  Добавить `uglify` задачи, используя приведенный ниже код.

    Задача уменьшает *combined.js* файл находится в каталоге temp и создает файл результатов в wwwroot/lib после стандартного соглашения об именовании  *\<имя файла\>. min.js*.
    
    ```javascript
    uglify: {
      all: {
        src: ['temp/combined.js'],
        dest: 'wwwroot/lib/combined.min.js'
      }
    },
    ```

11. В разделе grunt.loadNpmTasks() вызова, загружающий grunt contrib очистки включать одного вызова для jshint concat и uglify ниже коде.
    
    ```javascript
    grunt.loadNpmTasks('grunt-contrib-jshint');
    grunt.loadNpmTasks('grunt-contrib-concat');
    grunt.loadNpmTasks('grunt-contrib-uglify');
    ```

12. Сохранить *Gruntfile.js*. Файл должен выглядеть примерно в приведенном ниже примере.

    ![Пример файла grunt завершения](using-grunt/_static/gruntfile-js-complete.png)

13. Обратите внимание, что в списке задач Runner Explorer задач включает в себя `clean`, `concat`, `jshint` и `uglify` задачи. Выполнение каждой задачи в порядке и просмотрите результаты в обозревателе решений. Каждая задача должна выполняться без ошибок.
    
    ![Диспетчер выполнения задач выполнения каждой задачи](using-grunt/_static/task-runner-explorer-run-each-task.png)
    
    Создает новую задачу concat *combined.js* файл и помещает его в каталоге temp. Задача jshint просто работает и не создает выходных данных. Создает новую задачу uglify *combined.min.js* файл и помещает его в wwwroot/lib. По завершении решение должно выглядеть примерно как на снимке экрана ниже:
    
    ![После выполнения всех задач в обозревателе решений](using-grunt/_static/solution-explorer-after-all-tasks.png)
    
    > [!NOTE]
    > Дополнительные сведения о параметрах для каждого пакета [https://www.npmjs.com/](https://www.npmjs.com/) и поиск имени пакета, в поле поиска на главной странице. Например можно найти пакет grunt contrib очистки для получения документации ссылку, которая объясняет все его параметры.

### <a name="all-together-now"></a>Теперь все вместе

Используйте Grunt `registerTask()` метода для выполнения ряда задач в определенной последовательности. Например, для запуска этого примера, приведенные выше действия в том порядке, очистить "->" concat "->" jshint -> uglify, добавьте приведенный ниже код в модуль. Код должны быть добавлены как вызовы loadNpmTasks(), за пределами initConfig того же уровня.

```javascript
grunt.registerTask("all", ['clean', 'concat', 'jshint', 'uglify']);
```

Новая задача отображается в диспетчер выполнения задач в группе задач псевдоним. Правой кнопкой мыши и запустите его, так же, как и другие задачи. `all` Задача будет выполняться `clean`, `concat`, `jshint` и `uglify`, в порядке.

![псевдоним grunt задачи](using-grunt/_static/alias-tasks.png)

## <a name="watching-for-changes"></a>Контроль изменений

Объект `watch` задач следят файлов и каталогов. Контрольное значение автоматически запускает задачи при обнаружении изменений. Добавьте приведенный ниже код для initConfig для отслеживания изменений для \*JS-файлов в каталоге TypeScript. При изменении файла JavaScript `watch` будет выполняться `all` задачи.

```javascript
watch: {
  files: ["TypeScript/*.js"],
  tasks: ["all"]
}
```

Добавьте вызов `loadNpmTasks()` для отображения `watch` задач в Task Runner Explorer.

```javascript
grunt.loadNpmTasks('grunt-contrib-watch');
```

Щелкните правой кнопкой мыши задачу «Контрольные значения» в Task Runner Explorer и выберите в контекстном меню запуска. Командное окно, показывающий выполняемую задачу Контрольные значения будет отображаться «ожидание...» !". Откройте один из файлов TypeScript, добавьте пробел и затем сохраните файл. Это активация задачи контрольного значения и запускает другие задачи для выполнения в порядке. На следующем снимке экрана показано выполнения образца.

![под управлением выходных данных задачи](using-grunt/_static/watch-running.png)

## <a name="binding-to-visual-studio-events"></a>Привязка к событиям в Visual Studio

Если вы хотите запустить задач вручную каждый раз при работе в Visual Studio, можно привязать задач **перед построить**, **после построения**, **Очистить**, и  **Откройте проект** события.

Следует привязать `watch` , чтобы он запускался при каждом открытии Visual Studio. В Task Runner Explorer, щелкните правой кнопкой мыши задачи «Контрольные значения» и выберите **привязки > Открыть проект** в контекстном меню.

![Привязка задачи к Открытие проекта](using-grunt/_static/bindings-project-open.png)

Выгрузите и перезагрузите проект. При загрузке снова проекта, Контрольные значения начала задачи выполняются автоматически.

## <a name="summary"></a>Сводка

Grunt является runner мощные задачи, который может использоваться для большинства задач сборку клиента автоматизации. Grunt использует NPM для доставки пакетов и компонентов, интеграция с Visual Studio для работы с проектами. Visual Studio Task Runner Explorer обнаруживает изменения в файлах конфигурации и обеспечивает удобный интерфейс для выполнения задач, просмотр запущенных задач и привязывание задач к событиям в Visual Studio.
