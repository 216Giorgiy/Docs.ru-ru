## <a name="what-is-aspnet-web-pages"></a>Что такое веб-страницы ASP.NET

Веб-страницы ASP.NET — это платформа, можно использовать для создания динамических веб-страниц. Простой веб-страницы HTML является статическим. его содержимое определяется основных разметки HTML, который находится на странице. Динамических страниц, например тех, что создаются с помощью веб-страниц ASP.NET позволяют создать содержимое страницы на ходу, с помощью кода.

Динамические страницы позволяют выполнять множество различных операций. Можно запроса у пользователя ввод с помощью формы и затем изменить на странице отображается или его вид. Получить сведения от пользователя, сохраните его в базе данных и укажите ее позже. Можно отправить по электронной почте с веб-узла. Можно взаимодействовать с другими службами в Интернете (например, служба сопоставления) и создания страниц, интегрировать данные из этих источников.

## <a name="what-is-webmatrix"></a>Что такое WebMatrix

WebMatrix — это средство, которое интегрируется в редакторе веб-страницы, программа базы данных, веб-сервер для тестирования страниц и функций для публикации веб-сайта к Интернету. WebMatrix предоставляется бесплатно и его легко установить и простые в использовании. (Она также работает для простого HTML-страниц, а также для других технологий, как PHP).

Фактически не *имеют* использование WebMatrix для работы с веб-страниц ASP.NET. Можно создавать страницы с помощью текстового редактора, например и тестировать страницы с помощью веб-сервера, у вас есть доступ к. Тем не менее WebMatrix упрощает все очень просто, поэтому в этих учебниках будет использовать WebMatrix на протяжении.

## <a name="testing-the-page"></a>Тестирование страницы

В **файлы** рабочей области, щелкните правой кнопкой мыши *HelloWorld.cshtml* и нажмите кнопку **запуск в браузере**.

![Запуск страницы с помощью выполнения кнопки на ленте WebMatrix](getting-started/_static/image18.png)

WebMatrix запускает встроенных веб-сервера (IIS Express), можно использовать для тестирования страниц на компьютере. (Без IIS Express в WebMatrix, вы бы где-либо опубликовать страницу веб-сервере, прежде чем удалось проверить.) Откроется страница в браузере по умолчанию.

![&quot;Здравствуй, мир!&quot; страница, выполняемая в браузере](getting-started/_static/image19.png)

Обратите внимание, что при тестировании страницы в WebMatrix URL-адрес в браузере примерно `http://localhost:33651/HelloWorld.cshtml.` имя *localhost* ссылается на локальном сервере, то есть обслуживается, страницы веб-сервером, на локальном компьютере. Как уже отмечалось, в WebMatrix имеются программы веб-сервера с именем IIS Express, который выполняется при запуске страницы.

Число после *localhost* (например, *localhost:33651*) ссылается на *номер порта* на компьютере. Это число «канал», для этого конкретного веб-сайта IIS Express. Номер порта выбирается случайным образом из диапазона от 1024 до 65536 при создании веб-узла, и оно отличается для каждого сайта, который вы создаете. (Если вы тестируете свой собственный узел, номер порта почти наверняка будет другой номер, чем 33561.) С помощью другой порт для каждого веб-сайта, IIS Express можно хранить прямых которого взаимодействуют с веб-сайтов.

Позже при публикации сайта для общедоступного веб-сервера, больше нет *localhost* в URL-АДРЕСЕ. На этом этапе вы увидите их обычный URL-адрес вида `http://myhostingsite/mywebsite/HelloWorld.cshtml` или независимо от данной страницы. Более вы узнаете о публикации сайта в данной серии учебных курсов.

## <a name="this-is-what-aspnet-web-pages-are-about"></a>Это веб-страницах ASP.NET, о

Когда вы читаете, что веб-страниц ASP.NET создает динамический веб-содержимого, чем можно было видеть здесь является идея. Страница, которую вы только что создали содержит же разметки HTML, который вы уже видели. Он также может содержать код, который можно выполнять все виды задачи. В этом примере как непростая задача получения текущей даты и времени. Как видно, код можно соединять с HTML-код для вывода на странице. Когда кто-то запрашивает *.cshtml* страницу в браузере, ASP.NET обрабатывает страницы по-прежнему в руках веб-сервера. ASP.NET вставляет результат выполнения кода (если таковые имеются) в страницу как HTML. По завершении обработки кода ASP.NET отправляет полученную страницу в браузере. Все, что когда-либо возвращает браузер — HTML. Далее приведен:

![Концептуальный алгоритм как ASP.NET динамически создает HTML](getting-started/_static/image21.png)

Идея проста, но существует множество интересных задач, которые могут выполнять код, и существует множество интересных способов, в которых может динамически добавлять HTML-содержимого на страницу. И ASP.NET *.cshtml* страницы, как и любой HTML-страницы могут также включать код, который выполняется в браузере, сам (код JavaScript и jQuery). Все это в этот учебник набор и последующие рассматривается.
