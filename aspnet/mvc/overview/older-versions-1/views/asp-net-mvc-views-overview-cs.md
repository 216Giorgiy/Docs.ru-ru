## <a name="understanding-views"></a>Основные сведения о представлениях

Для страниц ASP или ASP.NET ASP.NET MVC не включает все, что соответствует непосредственно на страницу. В приложении ASP.NET MVC не страницы на диск, который соответствует пути в URL-адрес, введите в адресной строке браузера. -Это ближайший страницы в приложении ASP.NET MVC, то вызывается *представление*.

ASP.NET MVC браузера приложений, входящие запросы, сопоставляются с действия контроллера. Представление может возвратить следующие действия контроллера. Однако действия контроллера может выполнять действия, такие как перенаправление на другое действие контроллера другого типа.

Листинг 1 содержит простой контроллер с именем HomeController. HomeController предоставляет два действия контроллера, с именем Index() и Details().

**Листинг 1 - HomeController.cs**

[!code-csharp[Main](asp-net-mvc-views-overview-cs/samples/sample1.cs)]

Первое действие, действие Index(), можно вызвать, введя следующий URL-адрес в адресную строку браузера:

/ Home/Index

Второе действие, действие Details(), можно вызвать, введя этот адрес в адресную строку браузера:

-Home/подробности

Действие Index() возвращает представление. Большинство действий, которые вы создаете вернет представления. Тем не менее действие, которое может возвращать другие результаты действий. Например действие Details() возвращает RedirectToActionResult, который перенаправляет Index() действие входящего запроса.

Действие Index() содержит следующие одной строки кода:

View();

Эта строка кода возвращает представление, которое должен быть расположен по следующему пути на веб-сервере:

\Views\Home\Index.aspx

Путь к представлению выводится из имени контроллера и имя действия контроллера.

При желании можно явно задать, какие представления. Следующий код возвращает представление с именем Fred:

Представление (Fred);

При выполнении этой строки кода, представления возвращается из следующей папки:

\Views\Home\Fred.aspx

> [!NOTE] 
> 
> Если вы планируете создавать модульные тесты для приложения ASP.NET MVC является рекомендуется явно указывать имена представлений. Таким образом, можно создать модульный тест, чтобы убедиться, что ожидаемое представление возвращенный действия контроллера.
