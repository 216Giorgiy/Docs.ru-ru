## <a name="use-page-inspector-to-browse-to-a-view"></a>Использовать инспектор страниц в перейдите к представлению

В Visual Studio 2012, щелкнуть правой кнопкой мыши любое представление в своем проекте выберите **просмотреть в инспекторе страниц**, инспектор страниц будет выяснить, маршрута и отображения страницы.

В **обозревателе решений**, разверните **представления** папки и затем **Главная** папки. Щелкните правой кнопкой мыши файл Index.cshtml и выберите **просмотреть в инспекторе страниц**.

![Index.cshtml Просмотр в инспекторе страниц](using-page-inspector-in-aspnet-mvc/_static/image8.png)

По умолчанию инспектор страниц закреплен в окне слева от среды Visual Studio. При желании можно закрепить его в другом месте или открепить окно. В разделе [как: размещение и закрепление окон](https://msdn.microsoft.com/en-us/library/z4y0hsax.aspx).

В верхней области окна инспектора страниц показываются текущей страницы в окне браузера. На нижней панели отображаются страницы в разметке HTML, а также некоторые вкладки, где можно просмотреть различные аспекты страницы. На нижней панели аналогичен [средств разработчика F12](https://msdn.microsoft.com/en-us/ie/aa740478) в Internet Explorer.

![Приложение ASP.NET MVC в инспекторе страниц](using-page-inspector-in-aspnet-mvc/_static/image10.png)

В этом учебнике используется **HTML** и **стили** вкладки для быстрого перехода и внесения изменений в приложение.

## <a name="enableinspection-mode"></a>Режим EnableInspection

Чтобы перевести инспектор страниц в режиме инспектирования, нажмите кнопку **инспектировать** кнопки. В режиме инспектирования когда указатель мыши над любой частью подготовленной странице соответствующая разметка исходного кода выделяется.

![Переключиться в режим проверки](using-page-inspector-in-aspnet-mvc/_static/image12.png)

Теперь наведите курсор на различные части страницы в инспекторе страниц. После этого указатель мыши изменяется знака плюс и будет выделен под элементом:

![Наведении указателя мыши на div.content оболочки](using-page-inspector-in-aspnet-mvc/_static/image14.png)

При перемещении указателя мыши в Visual Studio выделяет соответствующий синтаксис Razor в исходном файле. Если HTML-элемент находится на другом исходном файле, Visual Studio автоматически открывает файл.

В инспекторе страниц **HTML** вкладке отображается HTML-код, который был создан с помощью синтаксиса Razor. При перемещении указателя мыши, выделяются HTML-элементов. **Стили** вкладке отображаются правила CSS для элемента.

## <a name="use-page-inspector-to-make-changes-to-markup"></a>Для внесения изменений в разметке использовать инспектор страниц

Инспектор страниц позволяет найти разметки, расположение которого могут быть неявными. Затем можно изменить разметку и результирующие изменения.

Чтобы увидеть его, нажмите кнопку **инспектировать** и прокрутите до нижней части страницы в окне инспектора страниц.

При перемещении указателя мыши в области нижнего колонтитула инспектор страниц будет открывать \_Layout.cshtml файл и выделяет раздел страницы макета, выбранный вами. Как видите, нижний колонтитул, определенные в файле макета, а не самого представления.

![Нижние колонтитулы](using-page-inspector-in-aspnet-mvc/_static/image16.png)

Теперь наведите указатель мыши на строку с авторских прав <a id="a"> </a>Обратите внимание. В \_страницу Layout.cshtml, соответствующая строка будет выделена.

![Выделенной об авторских правах строке нижнего колонтитула](using-page-inspector-in-aspnet-mvc/_static/image18.png)

Добавьте в конец строки в текст \_Layout.cshtml файл.

&lt;p&gt;&amp;копирование; @DateTime.Now.Year -Rocks Мое приложение ASP.NET MVC! &lt;/p&gt;

Сейчас нажмите клавиши Ctrl + Alt + Ввод или щелкните панель обновления, чтобы просмотреть результаты в окне браузера инспектора страниц.

![Мои Rocks приложения ASP.NET!](using-page-inspector-in-aspnet-mvc/_static/image20.png)

Вы могли подумать нижнего колонтитула, определенные в Index.cshtml, что она отключена в \_Layout.cshtml и найти он автоматически инспектора страниц.

## <a name="preview-css-changes-in-the-styles-window"></a>Просмотр изменений CSS в окне «Стили»

Далее можно использовать инспектор страниц **стили** окно для предварительного просмотра изменения CSS.

Нажмите кнопку **инспектировать** чтобы перевести инспектор страниц в режиме проверки.

В окне браузера инспектора страниц наведите указатель мыши на раздел «Домашняя страница» до **div.content оболочки** метка отображается.

![Наведении указателя мыши на div.content оболочки](using-page-inspector-in-aspnet-mvc/_static/image26.png)

Щелкните в разделе div.content оболочку, а затем наведите указатель мыши на **стили** окна. **Syles** окне показаны все правила CSS для этого элемента. Прокрутите вниз до селектор класса поиска .featured .content оболочки. Теперь снимите флажок для свойства цвет фона.

![Цвет фона, снимите флажок](using-page-inspector-in-aspnet-mvc/_static/image28.png)

Обратите внимание на то, как изменение Предварительный просмотр мгновенно в окне браузера инспектора страниц.

Снова установите флажок, затем дважды щелкните значение свойства и измените его на красный. Изменения немедленно показано:

![Красного цвета фона](using-page-inspector-in-aspnet-mvc/_static/image30.png)

**Стили** облегчает тестирование и предварительного просмотра CSS изменяется до фиксации изменений стиль делает окно листа сам.

<a id="css_auto_sync"></a>
## <a name="css-auto-sync"></a>Автосинхронизация CSS

> [!NOTE]
> Этот компонент требуется версия 1.3 инспектора страниц.


Функция автоматической синхронизации CSS позволяет напрямую изменить CSS-файл и увидеть изменения непосредственно в браузере инспектора страниц.

Нажмите кнопку **инспектировать** чтобы перевести инспектор страниц в режиме проверки.

В браузере инспектора страниц наведите указатель мыши на раздел «Домашняя страница» до **div.content оболочки** метка отображается. Щелкните один раз, чтобы выбрать этот элемент.

**Syles** окне показаны все правила CSS для этого элемента. Прокрутите вниз до селектор класса поиска .featured .content оболочки. Щелкните «.featured .content-программу-оболочку». Инспектор страниц открывает CSS-файл, определяющий этот стиль (Site.css) и выделяет соответствующего стиля CSS.

![](using-page-inspector-in-aspnet-mvc/_static/image32.png)

Теперь измените значение `background-color` для «red». Изменение немедленно отображается в браузере инспектора страниц.

![](using-page-inspector-in-aspnet-mvc/_static/image34.png)

<a id="css_color_picker"></a>
## <a name="using-the-css-color-picker"></a>В палитре цветов CSS

Редактор CSS в Visual Studio 2012 имеет палитру цветов, позволяющие легко выбрать и вставить цвета. Палитра содержит стандартную палитру цветов и поддерживает стандартные имена цветов, хэш-кодов, цветов RGB, RGBA, HSL и HSLA ведет список цветов, которые вы недавно использовали в документе.

В предыдущем разделе, вы изменили значение `background-color` свойства. Чтобы вызвать палитру цветов, поместите курсор после имени свойства и типа  **#**  или **rgb (**.

![Палитра цветов CSS](using-page-inspector-in-aspnet-mvc/_static/image36.png)

Щелкните цвет, выберите его и нажмите клавишу Стрелка вниз, а затем используйте клавиши со стрелками влево и вправо для перехода цвета. При посещении цвет, соответствующий шестнадцатеричное значение открывается для предварительного просмотра:

![значение свойства цвет фона предварительного просмотра](using-page-inspector-in-aspnet-mvc/_static/image38.png)

Если цветовой шкалы не имеет точного нужного цвета, можно использовать средство выбора цвета pop раскрывающегося списка. Чтобы открыть его, щелкните значок шеврона в правом конце цветовой шкалы или один или два раза нажмите клавишу Стрелка вниз на клавиатуре.

![Палитра цветов CSS Pop раскрывающегося списка](using-page-inspector-in-aspnet-mvc/_static/image40.png)

Выберите цвет из вертикальная черта справа. Здесь показан градиент цвета в главном окне. Выберите цвет непосредственно на вертикальную панель, нажав клавишу ВВОД или щелкнув любую из точек в главном окне выберите с большей точностью.

Существует ли цвет на экране компьютера, который вы хотите использовать (это не обязательно должно находиться в пользовательском интерфейсе Visual Studio), его значение можно записать с помощью пипетки в правом нижнем углу.

Также можно изменить степени непрозрачности цвета, переместив ползунок в нижней части палитру цветов. Таким образом, изменения цвета значения RGBA, так как формат RGBA может представлять непрозрачности.

После выбора цвета, нажмите клавишу ВВОД, а затем введите точку с запятой для завершения операции цвет фона в *Site.css* файла.

<a id="map_dynamic_elements"></a>
## <a name="mapping-dynamic-page-elements-to-javascript"></a>Сопоставление динамические элементы страниц в код JavaScript

В современных веб-приложений элементы на странице часто создаются динамически с помощью JavaScript. Это означает, что имеется не разметки static (HTML или Razor), соответствующий эти элементы страницы.

С версии 1.3 инспектор страниц теперь можно изменять элементы, добавленные динамически на страницу обратно в соответствующий код JavaScript. Чтобы продемонстрировать эту функцию, мы будем использовать [шаблона приложения одной странице (SPA)](../../../single-page-application/overview/introduction/knockoutjs-template.md).

> [!NOTE]
> Требуется шаблон SPA [ASP.NET и 2012.2 средства Web](https://go.microsoft.com/fwlink/?LinkId=282650) обновления.


В Visual Studio выберите **файл** &gt; **новый проект**. Слева разверните узел **Visual C#**выберите **Web**, а затем выберите **веб-приложение ASP.NET MVC4**. Нажмите кнопку **ОК**.

В **нового проекта ASP.NET MVC 4** диалогового окна выберите **одностраничного приложения**.

В обозревателе решений откройте **представления** папки и затем **Главная** папки. Щелкните правой кнопкой мыши файл Index.cshtml и выберите **просмотреть в инспекторе страниц**.

Первое, что именно отображается в браузере инспектора страниц — это страница входа в систему. Нажмите кнопку «Зарегистрироваться» и создайте имя пользователя и пароль. После регистрации, вход в приложение и создает список дел с некоторых элементов.

Нажмите кнопку **инспектировать** чтобы перевести инспектор страниц в режиме проверки. В браузере инспектора страниц щелкните один из элементов, задачу. Обратите внимание, что вместо выделены синим цветом, элемент выделен оранжевым цветом, с «JS» рядом с именем элемента. Это означает, что элемент был создан динамически при помощи сценария.

![](using-page-inspector-in-aspnet-mvc/_static/image44.png)

Кроме того, оранжевый подчеркивание отображается на **стек вызовов** вкладки. Это означает, что **стек вызовов** панель содержит дополнительные сведения об элементе.

Щелкните **стек вызовов** вкладки. **Стек вызовов** области отображается стек вызовов для вызова JavaScript, создавшего элемент. Вызывается на внешние библиотеки, например jQuery свернуты, легко увидеть вызовы к сценарию приложения.

![](using-page-inspector-in-aspnet-mvc/_static/image46.png)

Чтобы просмотреть полный стек, включая вызовы внешние библиотеки, можно развернуть в узлах «Внешние библиотеки»:

![](using-page-inspector-in-aspnet-mvc/_static/image48.png)

Если щелкнуть элемент в стеке вызовов, Visual Studio открывает файл кода и выделяет соответствующих сценариев.

![](using-page-inspector-in-aspnet-mvc/_static/image50.png)
