## <a name="application-structure"></a>Структура приложений

Горячий SPA полотенец предоставляет папке приложения, которая содержит JavaScript и HTML-файлов, которые определяют приложение.

В папке приложения:

- durandal
- службы
- ViewModels
- представления

Папка приложения содержит коллекцию модулей. Эти модули инкапсулируют функциональность и объявлять зависимости на другие модули. Папка views содержит HTML-код для приложения и папке viewmodels содержит логику представления для представлений (MVVM шаблоне общего). Папка служб идеально подходит для размещения любые общие службы, в приложении могут понадобиться, например извлечение данных HTTP или для взаимодействия локального хранилища. Часто бывает несколько viewmodels для повторного использования кода из модулей службы.

## <a name="how-do-i-build-on-hot-towel"></a>Как создавать на активном полотенец?

Просто приступите к добавлению кода!

1. Добавить собственные серверный код, предпочтительно Entity Framework и WebAPI (которых предстают Breeze.js)
2. Добавление представлений для `App/views` папки
3. Добавить viewmodels для `App/viewmodels` папки
4. Добавление нового представления HTML и частичной привязки данных
5. Обновление маршрутов навигации в `shell.js`
