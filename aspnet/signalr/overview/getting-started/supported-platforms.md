## <a name="server-system-requirements"></a>Требования к системе для сервера

Компонент сервера SignalR может размещаться на разных конфигураций сервера. В этом разделе описаны поддерживаемые версии операционных систем, .NET framework, сервер IIS и другие компоненты.

### <a name="supported-server-operating-systems"></a>Поддерживаемые серверные операционные системы

Компонент сервера SignalR можно поместить в следующих операционных систем сервера или клиента. Обратите внимание, что для SignalR, использовать WebSocket, Windows Server 2012, Windows Server 2016 или Windows 8 требуется (WebSocket можно использовать в Windows Azure веб-сайтов, пока равным сайта версии .NET framework 4.5 и веб-сокеты включен на сайте Страница конфигурации).

- Windows Server 2016
- Windows Server 2012
- Windows Server 2008 r2
- Windows 10
- Windows 8
- Windows 7
- Microsoft Azure

### <a name="supported-server-net-framework-version"></a>Поддерживаемые серверные версии платформы .NET Framework

SignalR 2 поддерживается только в .NET Framework 4.5. В разделе [рекомендуемых обновлений](#updates) раздела для обновления, которые повышают надежность, совместимости, стабильности и производительности.

### <a name="supported-server-iis-versions"></a>Поддерживаемых версиях IIS

Если SignalR размещается в службах IIS, поддерживаются следующие версии. Обратите внимание, что при использовании клиентской операционной системы, такие как для разработки приложений (Windows 8 или Windows 7) полной версии IIS или Cassini следует не использоваться, так как ограничение в 10 одновременных подключений, установленные, которых истекает очень быстро с момента подключения будет временной, часто повторно установить и они не удаляются сразу же после больше не используются. Следует использовать IIS Express в клиентских операционных системах.

Также Обратите внимание, для SignalR использовать WebSocket, следует использовать IIS 8 или IIS 8 Express, Windows 8, Windows Server 2012 или более поздней версии, необходимо использовать сервер и WebSocket необходимо включить в службах IIS. Сведения о включении WebSocket в службах IIS см. в разделе [поддержку протокола WebSocket служб IIS 8.0](https://www.iis.net/learn/get-started/whats-new-in-iis-8/iis-80-websocket-protocol-support).

- IIS 8 или IIS 8 Express.
- Службы IIS 7 и 7.5. Поддержка [без расширений URL-адреса](https://support.microsoft.com/kb/980368) является обязательным.
- IIS должна быть запущена в режиме интеграции. классический режим не поддерживается. Задержка сообщений до 30 секунд могут возникать из-Если запустить IIS в классическом режиме, с помощью транспорта Server-Sent события.
- Ведущее приложение должна быть запущена в режиме полного доверия.

## <a name="client-system-requirements"></a>Системные требования для клиента

SignalR может использоваться в различных клиентских платформ. В этом разделе описываются требования к системе для использования SignalR в веб-браузеры, настольных приложений Windows, приложений Silverlight и мобильных устройств.

### <a name="web-browsers"></a>Веб-браузеры

SignalR может использоваться в различных браузерах, но обычно поддерживаются только две последние версии.

Приложения, использующие SignalR в браузерах необходимо использовать версию jQuery 1.6.4 или основных более поздних версий (например 1.7.2, 1.8.2 или 1.9.1).

SignalR можно использовать в следующих браузерах:

- Версии Microsoft Internet Explorer 8, 9, 10 и 11. Современные, рабочего стола и мобильных устройств, поддерживаются.
- Mozilla Firefox: текущей версии - 1, версии, Mac и Windows.
- Google Chrome: текущую версию - 1, версии, Mac и Windows.
- Safari: текущую версию - 1, версии, Mac и iOS.
- Opera: текущая версия — 1, только для Windows.
- Браузера Android

Помимо некоторые браузеры, различных транспортов, которые использует SignalR имеют свои собственные требования. Следующих транспортов, поддерживаемых в следующих конфигурациях:

<a id="browser"></a>

**Требования к транспорта браузера в Интернете**

| Transport | Internet Explorer | Chrome (Windows или iOS) | Firefox | Safari (OSX или iOS) | Android |
| --- | --- | --- | --- | --- | --- |
| Протокол WebSocket | 10+ | Текущая - 1 | Текущая - 1 | Текущая - 1 | Н/Д |
| Сервер отправил события | Н/Д | Текущая - 1 | Текущая - 1 | Текущая - 1 | Н/Д |
| ForeverFrame | 8+ | Н/Д | Н/Д | Н/Д | 4.1 |
| Долго опрашивающего | 8+ | Текущая - 1 | Текущая - 1 | Текущая - 1 | 4.1 |

\*: требуется для обеспечения полной функциональности на 6 +.

#### <a name="unsupported-browsers"></a>Неподдерживаемые браузеры

Хотя SignalR *может* запустить без основных проблем в более старых версий браузера, мы не тестируйте активно SignalR их и обычно не исправляет ошибки, которые могут отображаться в них.

### <a name="windows-desktop-and-silverlight-applications"></a>Рабочий стол и приложения Silverlight

Наряду с выполнением в веб-браузере, SignalR может размещаться в автономный клиент Windows или приложениях Silverlight. Рабочий стол Windows и Silverlight SignalR приложения имеют следующие требования к системе.

- Приложений с помощью .NET 4, поддерживаются в Windows XP с пакетом обновления 3 или более поздней версии.
- Приложений с помощью .NET Framework 4.5, поддерживаются в Windows Vista или более поздней версии.

Помимо операционной системы и требования к .NET framework для SignalR транспорты имеют свои собственные требования. Следующих транспортов, поддерживаемых в следующих конфигурациях:

**Рабочий стол Windows и требования транспорта Silverlight**

| Transport | Приложения .NET | Silverlight |
| --- | --- | --- |
| Веб-сокеты | Windows 8 и более поздних и .NET 4.5 + | Н/Д |
| Рамки | Н/Д | Н/Д |
| Сервер отправил события | .NET 4 + | 5+ |
| Долго опрашивающего | .NET 4 + | 5+ |
