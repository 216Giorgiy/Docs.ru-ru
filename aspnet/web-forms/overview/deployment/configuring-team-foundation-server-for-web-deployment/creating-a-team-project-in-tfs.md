> В этом разделе описывается создание нового командного проекта в Team Foundation Server (TFS) 2010.


Этот раздел входит в состав серию учебников, исходя из требования к развертыванию enterprise вымышленная компания Fabrikam, Inc. Этот учебник ряд использует образец решения & #x 2014; [решения диспетчера контактов](../web-deployment-in-the-enterprise/the-contact-manager-solution.md)& #x 2014; для представления веб-приложения с реалистичных уровень сложности, включая приложения ASP.NET MVC 3, Windows Службы Communication Foundation (WCF) и проект базы данных.

## <a name="task-overview"></a>Общие сведения о задаче

Подготовить и использовать новый командный проект в TFS, необходимо выполнить следующие общие действия:

- Предоставить разрешения пользователю, который будет создан новый командный проект.
- Создайте командный проект.
- Предоставить разрешения членам команды, которые будут работать в проекте.
- Проверка некоторое содержимое.

В этом разделе будет показано, как для выполнения этих процедур, и будет идентифицировать пользователей и роли задания, которые могут отвечать за каждой процедуры. Имейте в виду, что, в зависимости от структуры организации, каждая из этих задач может быть ответственность за другое лицо.

Задачи и пошаговых инструкций в этом разделе предполагается, что установлен и настроен TFS, и вы создали коллекцию командных проектов в рамках процесса настройки. Дополнительные сведения об этих предположений и общие сведения о сценарии см. раздел [Настройка сервера сборки TFS для развертывания веб-](configuring-a-tfs-build-server-for-web-deployment.md).

## <a name="grant-permissions-to-the-team-project-creator"></a>Предоставление разрешений создатель командного проекта

Для создания нового командного проекта, потребуются следующие разрешения:

- Необходимо иметь **создавать новые проекты** разрешение на уровне приложений TFS. Обычно предоставить это разрешение, добавив пользователям **Администраторы коллекции проектов** группы TFS. **Администраторы Team Foundation** глобальной группы также включает это разрешение.
- Необходимо иметь разрешение на создание новых сайтов командных внутри коллекции сайтов SharePoint, который соответствует коллекции командного проекта TFS. Обычно его можно предоставить путем добавления пользователя в группу SharePoint с **полный доступ** права на SharePoint, семейство веб-сайтов.
- При использовании функции SQL Server Reporting Services, необходимо быть членом **Диспетчер содержимого Team Foundation** роли в службах Reporting Services.

### <a name="who-performs-these-procedures"></a>Кто выполняет эти процедуры?

Как правило у пользователя или группы, являющегося администратором развертывания TFS также выполняет эти процедуры.

Поскольку это привилегированным набор разрешений, новые командные проекты обычно создаются небольшое подмножество пользователей с ответственность за администрирование развертывания TFS. Разработчики не обычно предоставляются разрешения на создание нового командного проекта.

### <a name="grant-permissions-in-tfs"></a>Предоставление разрешений в Team Foundation Server

Если вы хотите разрешить пользователям создавать новые командные проекты, первая задача высокого уровня является добавление пользователю **Администраторы коллекции проектов** для коллекции командных проектов.

**Чтобы добавить пользователя в группу администраторов коллекции проектов**

1. На сервере TFS на **запустить** последовательно выберите пункты **все программы**, нажмите кнопку **Microsoft Team Foundation Server 2010**, а затем нажмите кнопку **Team Foundation Консоль администрирования**.
2. В дереве навигации, разверните **уровня приложения**, а затем нажмите кнопку **коллекций командных проектов**.

    ![](creating-a-team-project-in-tfs/_static/image1.png)
3. В **коллекций командных проектов** выберите коллекцию, которым требуется управлять командных проектов.

    ![](creating-a-team-project-in-tfs/_static/image2.png)
4. На **Общие** щелкните **членство в группе**.

    ![](creating-a-team-project-in-tfs/_static/image3.png)
5. В **глобальных групп** выберите **Администраторы коллекции проектов** группы, а затем нажмите кнопку **свойства**.
6. В **свойства группы Team Foundation Server** выберите **пользователя или группы Windows**, а затем нажмите кнопку **добавить**.

    ![](creating-a-team-project-in-tfs/_static/image4.png)
7. В **Выбор пользователей, компьютеров или групп** диалоговое окно, введите имя пользователя, необходимо иметь возможность создавать новые командные проекты, щелкните **проверить имена**, а затем нажмите кнопку **ОК** .

    ![](creating-a-team-project-in-tfs/_static/image5.png)
8. В **свойства группы Team Foundation Server** диалоговое окно, нажмите кнопку **ОК**.
9. В **глобальных групп** диалоговое окно, нажмите кнопку **закрыть**.

### <a name="grant-permissions-in-sharepoint-services"></a>Предоставление разрешений в службах SharePoint

Далее необходимо предоставить пользователю соответствующие разрешения для создания новых сайтов командных коллекции веб-сайта SharePoint, который соответствует коллекции командного проекта TFS.

**Чтобы предоставить разрешения на полный доступ для коллекции сайтов SharePoint**

1. В консоли администрирования Team Foundation Server на **коллекций командных проектов** выберите коллекции командных проектов, которым требуется управлять.
2. На **сайт SharePoint** запомните значение **текущее местоположение сайта по умолчанию** URL-адрес.

    ![](creating-a-team-project-in-tfs/_static/image6.png)
3. Откройте Internet Explorer, а затем перейдите по URL-адресу, указанных в шаге 2.

    > [!NOTE]
    > Если вы не вошли в Windows как пользователь, создавший коллекции командных проектов, необходимо выполнить вход в SharePoint от имени этого пользователя для продолжения.
4. На **действия сайта** меню, нажмите кнопку **параметры сайта**.

    ![](creating-a-team-project-in-tfs/_static/image7.png)
5. На **параметры сайта** в разделе **пользователей и разрешений**, нажмите кнопку **пользователей и групп**.
6. На панели навигации слева щелкните элемент **группы**.

    ![](creating-a-team-project-in-tfs/_static/image8.png)
7. На **пользователи и группы: все группы** щелкните **настроить группы для этого сайта**.

    ![](creating-a-team-project-in-tfs/_static/image9.png)

    > [!NOTE]
    > Может появиться **HTTP 404 не найдено** ошибка из-за двойной ошибки кодирования HTTP. В этом случае замените URL-адрес с этим:   
    > [*URL-адрес коллекции сайта*] /\_layouts/permsetup.aspx  
    > Пример:  
    > http://tfs/sites/Fabrikam%20Web%20Projects/\_layouts/permsetup.aspx
8. На **настроить группы для этого сайта** страницы, добавьте пользователя, который будет создавать командные проекты для **владельцев** группы, а затем нажмите кнопку **ОК**.

    ![](creating-a-team-project-in-tfs/_static/image10.png)

Дополнительные сведения о включении пользователям создавать новые командные проекты в коллекции командных проектов см. в разделе [Установка прав администратора для коллекций командных проектов](https://msdn.microsoft.com/library/dd547204.aspx).

## <a name="create-a-new-team-project-and-add-users"></a>Создание нового командного проекта и добавление пользователей

При наличии необходимых разрешений можно использовать **Team Explorer** окна в Visual Studio 2010 для создания нового командного проекта. Такой подход предоставляет мастер, который собирает всю необходимую информацию и выполняет необходимые задачи в TFS, SharePoint и SQL Server Reporting Services. Также необходимо предоставить разрешения на новый командный проект, чтобы члены команды разработчиков, чтобы включить их для добавления и изменения содержимого.

### <a name="who-performs-these-procedures"></a>Кто выполняет эти процедуры?

Обычно администратор TFS или руководитель группы разработчиков, выполняет эти процедуры.

### <a name="create-a-new-team-project"></a>Создание нового командного проекта

Далее описано, как для создания нового командного проекта в Team Foundation Server 2010.

**Для создания нового командного проекта**

1. На **запустить** последовательно выберите пункты **все программы**, нажмите кнопку **Microsoft Visual Studio 2010**, щелкните правой кнопкой мыши **Microsoft Visual Studio 2010**, затем **Запуск от имени администратора**.

    > [!NOTE]
    > Если не запустить Visual Studio 2010 с правами администратора, мастер нового командного проекта на последний шаг не удастся.
2. Если **контроль учетных записей пользователей** диалоговое окно, нажмите кнопку **Да**.
3. В Visual Studio на **команды** меню, нажмите кнопку **подключиться к Team Foundation Server**.

    > [!NOTE]
    > Если вы уже настроили соединение с сервером TFS, можно пропустить шаги 4 – 7.
4. В **подключение к командному проекту** диалоговое окно, нажмите кнопку **серверов**.
5. В **Добавление или удаление Team Foundation Server** диалоговое окно, нажмите кнопку **добавить**.
6. В **Добавление Team Foundation Server** диалоговое окно «» укажите сведения об экземпляре TFS и нажмите кнопку **ОК**.

    ![](creating-a-team-project-in-tfs/_static/image11.png)
7. В **Добавление или удаление Team Foundation Server** диалоговое окно, нажмите кнопку **закрыть**.
8. В **подключение к командному проекту** установите флажок экземпляр TFS, необходимо подключиться, выберите команду проект коллекции, которые необходимо добавить, а затем нажмите кнопку **Connect**.

    ![](creating-a-team-project-in-tfs/_static/image12.png)
9. В **Team Explorer** окно, щелкните правой кнопкой мыши команда коллекции проектов, а затем нажмите кнопку **нового командного проекта**.

    ![](creating-a-team-project-in-tfs/_static/image13.png)
10. В **нового командного проекта** диалоговое окно, введите имя и описание для командного проекта и нажмите кнопку **Далее**.

    > [!NOTE]
    > Если командный проект содержит пробелы, могут возникнуть некоторые проблемы при выполнении использовать средство Internet Information Services (IIS) веб-развертывания (Web Deploy) для развертывания пакетов из выходной путь. Пробелы в пути можно сделать гораздо труднее, для выполнения команд веб-развертывания.

    ![](creating-a-team-project-in-tfs/_static/image14.png)
11. На **Выбор шаблона процесса** выберите шаблон процесса, который вы хотите использовать для управления процессом разработки и нажмите кнопку **Далее**.

    > [!NOTE]
    > Дополнительные сведения о шаблонах процессов TFS. в разделе [шаблонов процессов и средств](https://msdn.microsoft.com/vstudio/aa718795).
12. На **параметры сайта группы** оставьте параметры по умолчанию без изменений и нажмите кнопку **Далее**.
13. Этот параметр создает или определяет сайт группы SharePoint, который связан с командным проектом TFS. Команды разработчиков могут использовать этот сайт для управления документации, участвовать в цепочки обсуждений, создавать вики-страницы и выполнять различные задачи, которые не связаны с кодом. Дополнительные сведения см. в разделе [взаимодействий между продуктов SharePoint и Team Foundation Server](https://msdn.microsoft.com/library/ms253177.aspx).
14. На **укажите параметры системы управления версиями** оставьте параметры по умолчанию без изменений и нажмите кнопку **Далее**.
15. Этот параметр определяет или создает папку в иерархии папок TFS, который будет выступать в качестве корневой папки для содержимого.
16. На **подтверждение настроек командного проекта** щелкните **Готово**.
17. Когда новый командный проект создан успешно, на **командный проект создан** щелкните **закрыть**.

### <a name="add-users-to-a-team-project"></a>Добавление пользователей в командный проект

После создания нового командного проекта можно предоставить разрешения для пользователей, чтобы включить их начать добавление и совместной работы с содержимым.

**Чтобы добавить пользователей в командный проект**

1. В Visual Studio 2010 в **Team Explorer** , щелкните правой кнопкой мыши командный проект, подведите указатель к **параметры командного проекта**, а затем нажмите кнопку **членство в группе**.

    ![](creating-a-team-project-in-tfs/_static/image15.png)
2. Чтобы дать пользователю возможность добавлять, изменять и удалять кода в системе управления версиями, необходимо добавить его к **участники** группы.
3. В **группы проекта** выберите **участники** группы, а затем нажмите кнопку **свойства**.

    ![](creating-a-team-project-in-tfs/_static/image16.png)
4. В **свойства группы Team Foundation Server** выберите **пользователя или группы Windows**, а затем нажмите кнопку **добавить**.

    ![](creating-a-team-project-in-tfs/_static/image17.png)
5. В **Выбор пользователей, компьютеров или групп** диалоговое окно, введите имя пользователя, которого нужно добавить в командный проект, нажмите кнопку **проверить имена**, а затем нажмите кнопку **ОК**.

    ![](creating-a-team-project-in-tfs/_static/image18.png)
6. В **свойства группы Team Foundation Server** диалоговое окно, нажмите кнопку **ОК**.
7. В **группы проекта** диалоговое окно, нажмите кнопку **закрыть**.
