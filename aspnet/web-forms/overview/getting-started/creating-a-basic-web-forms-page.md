<a name="creating-a-basic-aspnet-45-web-forms-page-in-visual-studio-2013"></a>Создание основных ASP.NET 4.5 Web Forms страницы в Visual Studio 2013
====================
по [Эрик Reitan](https://github.com/Erikre)

В этом пошаговом руководстве предоставляются общие сведения о среде веб-разработки в [Microsoft Visual Studio 2013](https://www.microsoft.com/visualstudio/11/downloads#vs) и [Microsoft Visual Studio Express 2013 для Web](https://www.microsoft.com/visualstudio/11/downloads#express-web). Это пошаговое руководство поможет создать простой страницы веб-форм ASP.NET и иллюстрирует основные приемы создания новой страницы, добавление элементов управления и написания кода.

В данном пошаговом руководстве представлены следующие задачи.

- Создание проекта приложения веб-форм файл системы.
- Знакомство с Visual Studio.
- Создание страницы ASP.NET.
- Добавление элементов управления.
- Добавление обработчиков событий.
- Запуск и тестирование страницы из Visual Studio.

## <a name="prerequisites"></a>Предварительные требования


Для выполнения данного пошагового руководства требуется:

- [Microsoft Visual Studio 2013](https://www.microsoft.com/visualstudio/11/downloads#vs) или [Microsoft Visual Studio Express 2013 для Web](https://www.microsoft.com/visualstudio/11/downloads#express-web). Платформа .NET Framework устанавливается автоматически. 

    > [!NOTE] 
    > 
    > Microsoft Visual Studio 2013 и Microsoft Visual Studio Express 2013 для Web будет часто называться Visual Studio на протяжении этого учебника ряда.  
    >   
    > При использовании Visual Studio это пошаговое руководство предполагает, что выбрана **веб-разработки** набор параметров при первом запуске Visual Studio. Дополнительные сведения см. в разделе [как: выберите параметры среды веб-разработки](https://msdn.microsoft.com/library/ff521558.aspx).


## <a name="creating-a-web-application-project-and-a-page"></a>Создание проекта веб-приложения и страницы

<a id="sectionToggle0"></a>

В этой части пошагового руководства будет создать проект веб-приложения и добавление новой страницы. Будет также добавляется текст HTML и запустите страницу в браузере.

### <a name="to-create-a-web-application-project"></a>Чтобы создать проект веб-приложения

1. Откройте Microsoft Visual Studio.
2. В меню **Файл** выберите **Создать проект**.  
    ![Меню "файл"](creating-a-basic-web-forms-page/_static/image1.png)

    Откроется диалоговое окно **Новый проект** .
3. Выберите **шаблоны**  - &gt; **Visual C#**  - &gt; **Web** группа шаблонов в левой части экрана.
4. Выберите **веб-приложение ASP.NET** шаблона в центральном столбце.
5. Присвойте проекту имя ***BasicWebApp*** и нажмите кнопку **ОК** кнопки.   
![Диалоговое окно нового проекта](creating-a-basic-web-forms-page/_static/image2.png)
6. Затем выберите **Web Forms** шаблона и нажмите кнопку **ОК** кнопку, чтобы создать проект.  
![Диалоговое окно Новый проект ASP.NET](creating-a-basic-web-forms-page/_static/image3.png)  

    Visual Studio создает новый проект, включающий стандартные функциональные возможности на основе шаблона веб-форм. Не только предоставляет вам *Home.aspx* страницы, *About.aspx* страницы, *Contact.aspx* страницы, но также включает функции членства, который регистрирует пользователей и сохраняет учетные данные, чтобы они могут войти в веб-сайта. При создании новой страницы по умолчанию Visual Studio отображается страница в **источника** представление, где можно просмотреть элементы HTML страницы. На следующем рисунке показано, что бы в **источника** просмотра при создании нового веб-страницы с именем *BasicWebApp.aspx*.  
    ![Просмотр исходного кода](creating-a-basic-web-forms-page/_static/image4.png)

### <a name="creating-a-new-aspnet-web-forms-page"></a>Создание нового страница веб-форм ASP.NET


При создании нового приложения веб-форм с помощью **веб-приложение ASP.NET** шаблон проекта Visual Studio добавляет страницы ASP.NET (страницы веб-форм) с именем *Default.aspx*, также как несколько файлов и папки. Можно использовать *Default.aspx* страницу в качестве домашней страницы для веб-приложения. Однако в данном пошаговом руководстве будет создать и работать с новой страницы.

### <a name="to-add-a-page-to-the-web-application"></a>Добавление страницы в веб-приложение


1. Закрыть *Default.aspx* страницы. Чтобы сделать это, перейдите на вкладку, отображает имя файла и нажмите кнопку «Закрыть».
2. В **обозревателе решений**, щелкните правой кнопкой мыши имя веб-приложения (в этом учебнике, имя приложения — **BasicWebSite**), а затем нажмите кнопку **добавить**  - &gt; **Новый элемент**.   
Откроется диалоговое окно **Добавление нового элемента**.
3. Выберите **Visual C#**  - &gt; **Web** группа шаблонов в левой части экрана. Выберите **веб-форму** из середины списка и назовите его *FirstWebPage.aspx*.   
    ![Добавить новый элемент-диалоговое окно](creating-a-basic-web-forms-page/_static/image6.png)
4. Нажмите кнопку **добавить** для добавления в проект веб-странице.  
Visual Studio создаст новую страницу и открывает его.


### <a name="adding-html-to-the-page"></a>Добавление на страницу HTML


В этой части пошагового руководства вы добавите статический текст на страницу.

### <a name="to-add-text-to-the-page"></a>Добавление текста на страницу


1. В нижней части окна документа, щелкните **разработки** для перехода к **разработки** представления.

    Режим конструктора отображает текущую страницу WYSIWYG-подобном способом. На этом этапе у вас текст и элементы управления на странице, поэтому страницы пустыми, за исключением пунктирная линия, описывающий прямоугольник. Этот прямоугольник представляет **div** элемента на странице.
2. Щелкните внутри прямоугольника, описываются пунктирной линией.
3. Тип **Добро пожаловать в Visual Web Developer** и нажмите клавишу **ввод** дважды.

    На следующем рисунке показан текст, введенный в **разработки** представления.

    ![Текст приветствия в режиме конструктора](creating-a-basic-web-forms-page/_static/image7.png "текст приветствия в режиме конструктора")
4. Переключитесь в **источника** представления.

    Вы увидите HTML в **источника** представления, созданного во время ввода **разработки** представления.  
    ![Веб-страницы со статическим текстом](creating-a-basic-web-forms-page/_static/image8.png)


### <a name="running-the-page"></a>Запуск страницы


Прежде чем продолжить путем добавления элементов управления на страницу, можно запустить его.

### <a name="to-run-the-page"></a>Чтобы запустить страницу


1. В **обозревателе решений**, щелкните правой кнопкой мыши *FirstWebPage.aspx* и выберите **задать в качестве начальной страницы**.
2. Нажмите клавишу **CTRL + F5** для запуска страницы.

    В браузере отображается страница. Несмотря на то, что страница, вы создали имеет расширение имени файла *.aspx*, в данный момент работает подобно любой HTML-страницы.

    Чтобы отобразить страницу в браузере вы можете щелкнуть правой кнопкой страницы в **обозреватель решений** и выберите **Просмотр в браузере**.
3. Закройте браузер, чтобы остановить веб-приложения.


## <a name="adding-and-programming-controls"></a>Добавление и программирование элементов управления


<a id="sectionToggle1"></a>

Теперь добавим серверных элементов управления на страницу. Серверные элементы управления, такие как кнопки, метки, текстовые поля и другие знакомые элементы управления предоставляют стандартные функции обработки форм для ваших страниц веб-форм. Тем не менее можно создавать элементы управления с кодом, который выполняется на сервере, а не на клиенте.

Вы добавите [кнопку](https://msdn.microsoft.com/library/system.web.ui.webcontrols.button.aspx) управления, [текстовое поле](https://msdn.microsoft.com/library/system.web.ui.webcontrols.textbox.aspx) элемента управления и [метки](https://msdn.microsoft.com/library/system.web.ui.webcontrols.label.aspx) на страницу элемент управления и напишите код для обработки [щелкните](https://msdn.microsoft.com/library/system.web.ui.webcontrols.button.click.aspx) событий для [кнопку](https://msdn.microsoft.com/library/system.web.ui.webcontrols.button.aspx) элемента управления.

### <a name="to-add-controls-to-the-page"></a>Добавление элементов управления на страницу


1. Нажмите кнопку **разработки** для перехода к **разработки** представления.
2. Поместите курсор в конце **Добро пожаловать в Visual Web Developer** текст и нажмите клавишу **ввод** пять или более раз, чтобы создать дополнительное пространство в **div** поле элемента.
3. В **элементов**, разверните **Стандартная** группы, если он еще не развернут.  
Обратите внимание, что может потребоваться развернуть **элементов** окна слева, чтобы просмотреть его.
4. Перетащите [TextBox](https://msdn.microsoft.com/library/system.web.ui.webcontrols.textbox.aspx) управления на страницу и поместите его посередине **div** поле элемента, который имеет **Добро пожаловать в Visual Web Developer** в первой строке.
5. Перетащите [кнопку](https://msdn.microsoft.com/library/system.web.ui.webcontrols.button.aspx) управления на страницу и разместите его справа от [TextBox](https://msdn.microsoft.com/library/system.web.ui.webcontrols.textbox.aspx) элемента управления.
6. Перетащите [метка](https://msdn.microsoft.com/library/system.web.ui.webcontrols.label.aspx) управления на страницу и поместите его на отдельной строке под [кнопку](https://msdn.microsoft.com/library/system.web.ui.webcontrols.button.aspx) элемента управления.
7. Установите точку вставки выше [TextBox](https://msdn.microsoft.com/library/system.web.ui.webcontrols.textbox.aspx) управления, а затем введите **введите свое имя:** .

    Этот статический текст HTML используется заголовок [TextBox](https://msdn.microsoft.com/library/system.web.ui.webcontrols.textbox.aspx) элемента управления. Можно комбинировать статическим HTML и серверные элементы управления на одной странице. На следующем рисунке показано, как отображаются три элемента управления в **разработки** представления.

    ![Три элемента управления в режиме конструктора](creating-a-basic-web-forms-page/_static/image9.png "три элемента управления в режиме конструктора")


### <a name="setting-control-properties"></a>Установка свойств элемента управления


Visual Studio предлагает различные способы установки свойств элементов управления на странице. В этой части пошагового руководства будут заданы свойства в обеих **разработки** представление и **источника** представления.

### <a name="to-set-control-properties"></a>Задание свойств элемента управления


1. Во-первых, отображения **свойства** windows, выбрав из **представление** меню -&gt; **другие окна**  - &gt; **Окна свойств**. Можно также выбрать **F4** для отображения **свойства** окна.
2. Выберите [кнопку](https://msdn.microsoft.com/library/system.web.ui.webcontrols.button.aspx) управления, а затем в **свойства** задайте значение **текст** для **отображаемое имя**. Введенный текст отображается на кнопке в конструкторе, как показано на следующем рисунке.

    ![Задать текст кнопки](creating-a-basic-web-forms-page/_static/image10.png "кнопку текста")
3. Переключитесь в **источника** представления.

    **Источник** представление отображает HTML для страницы, включая элементы, создаваемые в Visual Studio для серверных элементов управления. Объявления элементов управления схожий с HTML синтаксис **asp:** и включает атрибут **runat =&quot;сервера&quot;**.

    Свойства элемента управления объявляются как атрибуты. Например, если значение [текст](https://msdn.microsoft.com/library/system.web.ui.webcontrols.button.text.aspx) свойство для [кнопку](https://msdn.microsoft.com/library/system.web.ui.webcontrols.button.aspx) управления, на шаге 1 фактически устанавливается **текст** атрибута в разметке элемента управления.

    > [!NOTE] 
    > 
    > Все элементы управления находятся внутри **формы** элемент, который также имеет атрибут **runat =&quot;сервера&quot;**. **Runat =&quot;сервера&quot;**  атрибута и **asp:** префикс элемента управления элементов управления, чтобы они обрабатываются в ASP.NET на сервере при запуске страницы. Код вне элементов **&lt;формы с runat =&quot;сервера&quot; &gt;** и **&lt;скрипт runat =&quot;сервера&quot; &gt;** передается без изменений в браузер, поэтому код ASP.NET должны находиться внутри элемента, в открывающем теге которого содержит **runat =&quot;сервера&quot;**  атрибута.
4. Далее можно добавить дополнительное свойство в [метка](https://msdn.microsoft.com/library/system.web.ui.webcontrols.label.aspx) элемента управления. Поместите точку вставки непосредственно после **asp: Label** в **&lt;asp: Label&gt;** тега, а затем нажмите клавишу **пробел**.

    Отображается список раскрывающегося списка, отображающий список доступных свойств, задаваемых для [метка](https://msdn.microsoft.com/library/system.web.ui.webcontrols.label.aspx) элемента управления. Эта функция называется **IntelliSense**, поможет вам в **источника** представление с синтаксисом серверных элементов управления, HTML-элементов и других элементов на странице. На следующем рисунке показана **IntelliSense** раскрывающегося списка для [метка](https://msdn.microsoft.com/library/system.web.ui.webcontrols.label.aspx) элемента управления.

    ![Атрибуты IntelliSense](creating-a-basic-web-forms-page/_static/image11.png "атрибуты IntelliSense")
5. Выберите **ForeColor** , а затем введите знак равенства.

    IntelliSense отображает список цветов.

    > [!NOTE] 
    > 
    > Можно отобразить **IntelliSense** раскрывающегося списка в любое время, нажав клавишу **CTRL + J** при просмотре кода.
6. Выберите цвет для **[метка](https://msdn.microsoft.com/library/system.web.ui.webcontrols.label.aspx)** текст элемента управления. Убедитесь, что выбор цвета, Темная, для чтения на белом фоне.

    **ForeColor** атрибут выполнен в цвет, выбранный вами, включая знак закрывающей кавычки.


### <a name="programming-the-button-control"></a>Программирование элемента управления Button


В этом пошаговом руководстве будет написан код, который считывает имя, что пользователь вводит в текстовое поле, а затем отображает имя в [метка](https://msdn.microsoft.com/library/system.web.ui.webcontrols.label.aspx) элемента управления.

### <a name="add-a-default-button-event-handler"></a>Добавьте обработчик событий по умолчанию


1. Переключитесь в **разработки** представления.
2. Дважды щелкните [кнопку](https://msdn.microsoft.com/library/system.web.ui.webcontrols.button.aspx) элемента управления.

    По умолчанию Visual Studio переключается в файл кода и создаст структуру обработчика для [кнопку](https://msdn.microsoft.com/library/system.web.ui.webcontrols.button.aspx) события элемента управления по умолчанию, [щелкните](https://msdn.microsoft.com/library/system.web.ui.webcontrols.button.click.aspx) событий. Файл кода программной отделяет разметку пользовательского интерфейса (например, HTML) в коде сервера (например, C#).   
   Курсор располагается добавлен код для этого обработчика событий.

    > [!NOTE]

### <a name="running-the-page"></a>Запуск страницы


Теперь можно проверить серверные элементы управления на странице.

### <a name="to-run-the-page"></a>Чтобы запустить страницу


1. Нажмите клавишу **CTRL + F5** для запуска страницы в браузере. Если возникает ошибка, повторно проверьте действия, перечисленные выше.
2. Введите имя в текстовое поле и нажмите кнопку **отображаемое имя** кнопки.

    Введенное имя отображается в [метка](https://msdn.microsoft.com/library/system.web.ui.webcontrols.label.aspx) элемента управления. Обратите внимание, что при нажатии кнопки страница отправляется на веб-сервер. Затем ASP.NET заново создает страницу, выполняет код (в этом случае [кнопку](https://msdn.microsoft.com/library/system.web.ui.webcontrols.button.aspx) элемента управления [щелкните](https://msdn.microsoft.com/library/system.web.ui.webcontrols.button.click.aspx) выполняется обработчик событий), а затем отправляет новую страницу в браузере. Если смотреть строки состояния в браузере появится страницы что кругового пути на веб-сервер при каждом нажатии кнопки.
3. В браузере, просмотрите исходный код страницы, вы используете, щелкнув правой кнопкой мыши на странице и выбрав **Просмотр исходного кода**.

    В исходном коде страницы отображается HTML без кода сервера. В частности, вы не видите **&lt;asp:&gt;** элементы, которые вы работали с **источника** представления. При выполнении страницы ASP.NET обрабатывает серверные элементы управления и отображает HTML-элементы на страницу, которые выполняют функции, представления элемента управления. Например **&lt;asp: Button&gt;** управления подготавливается к просмотру как HTML **&lt;тип входных данных =&quot;отправить&quot; &gt;** элемент.
4. Закройте браузер.

3. В панели смарт-тегов выберите **Автоформат**.

    **Автоформат** диалоговое окно, что позволяет выбрать схему форматирования календаря. На следующем рисунке показана **Автоформат** диалоговое окно для [календаря](https://msdn.microsoft.com/library/system.web.ui.webcontrols.calendar.aspx) элемента управления.
