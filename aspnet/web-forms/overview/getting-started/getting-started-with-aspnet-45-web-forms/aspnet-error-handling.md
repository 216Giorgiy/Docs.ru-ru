## <a name="overview"></a>Обзор

Приложения ASP.NET необходимо обрабатывать ошибки, возникающие при выполнении согласованным образом. ASP.NET использует среду CLR, который предоставляет способ уведомления приложений об ошибках единым образом. Когда происходит ошибка, создается исключение. Исключение — это любой ошибки, условие или неожиданное поведение, которое обнаруживает приложения.

В .NET Framework исключение — это объект, наследуемый от класса `System.Exception`. Исключение создается из области кода, где произошла проблема. Исключение передается вверх по стеку вызовов в место, где приложение предоставляет код для обработки исключения. Если приложение не обрабатывает исключение, браузер принудительно отобразить сведения об ошибке.

Рекомендуется, обработки ошибок в на уровне кода в `Try` / `Catch` / `Finally` блоки в коде. Попробуйте разместить эти блоки, чтобы пользователь мог устранить неполадки в контексте, в котором они присутствуют. Блоки обработки ошибок, слишком далеко от возникновения становится более сложным предоставить пользователям сведения, необходимые для устранения проблемы.

### <a name="exception-class"></a>Класс исключений

Класс исключений является базовым классом, от которого наследуются исключения. Большинство объектов исключения являются экземплярами некоторого класса, производного от класса Exception, таких как `SystemException` класса `IndexOutOfRangeException` класс, или `ArgumentNullException` класса. Класс Exception имеет свойства, такие как `StackTrace` свойства `InnerException` свойство и `Message` свойства, которые предоставляют подробные сведения о возникшей ошибке.

### <a name="exception-inheritance-hierarchy"></a>Иерархия наследования исключений

Среда выполнения имеет базовый набор исключений, производных от `SystemException` класс, который среда выполнения создает исключение при обнаружении исключения. Большинство классов, наследующих от класса Exception, таких как `IndexOutOfRangeException` класса и `ArgumentNullException` класса, не следует реализовывать дополнительные члены. Таким образом наиболее важные сведения об исключении можно найти в иерархии исключений, в имени исключения и сведения, содержащиеся в исключении.

### <a name="exception-handling-hierarchy"></a>Иерархия обработка исключений

В приложении веб-форм ASP.NET исключения могут быть обработаны на основе конкретных обработки иерархии. Исключение может быть обработано на следующих уровнях:

- Уровень приложения
- На уровне страниц
- Уровень кода

Когда приложение обрабатывает исключения, Дополнительные сведения об исключении, которое наследуется от класса Exception, можно часто извлекается и отображается для пользователя. Помимо приложения, страницы и уровне кода также может обрабатывать исключения, на уровне модуля HTTP и с помощью пользовательского обработчика в IIS.

Если блок catch не найден, среда CLR отображает пользователю сообщение о необработанном исключении и останавливает выполнение программы.

### <a name="safe-error-messages"></a>Безопасные сообщения об ошибках

Это **Обратите внимание** , когда приложение отображает сообщения об ошибках, он не должны содержать сведения, которые пользователь-злоумышленник может помочь атаковать приложения. Например если приложения попытка записи базе данных, оно не должно отобразиться сообщение об ошибке, включающее имя пользователя, им. По этой причине для пользователя отображается сообщение об ошибке общего красным цветом. Все дополнительные сведения отображаются только для разработчика на локальном компьютере.

## <a name="using-elmah"></a>Использование ELMAH

ELMAH (модули ведения журнала ошибок и обработчики) является средством ведения журнала ошибок, можно подключить приложение ASP.NET, как пакет NuGet. ELMAH предоставляет следующие возможности:

- Ведение журнала необработанных исключений.
- Веб-страницы, чтобы просмотреть весь журнал целиком записанный необработанных исключений.
- Веб-страницы, чтобы просмотреть подробные сведения о каждом журнал исключение.
- Уведомление по электронной почте каждой ошибки во время его появлении.
- RSS-канал последние 15 ошибок из журнала.

Перед началом работы с ELMAH, его необходимо установить. Это легко с помощью *NuGet* пакет установщика. Как упоминалось ранее в этой серии учебника NuGet является расширение Visual Studio, упрощающий установку и обновление библиотек с открытым исходным кодом и средств в Visual Studio.

1. В среде Visual Studio из **средства** последовательно выберите пункты **диспетчер пакетов библиотеки**  - &gt; **управление пакетами NuGet для решения**. 

    ![Обработка ошибок ASP.NET — управление пакетами NuGet для решения](aspnet-error-handling/_static/image6.png)
2. **Управление пакетами NuGet** диалоговое окно в среде Visual Studio.
3. В **управление пакетами NuGet** диалогового окна разверните **Online** слева, а затем выберите **nuget.org**. Затем найти и установить **ELMAH** пакета из списка доступных пакетов в интерактивном режиме. 

    ![Обработка ошибок ASP.NET - ELMA пакета NuGet](aspnet-error-handling/_static/image7.png)
4. Необходимо иметь подключение к Интернету для загрузки пакета.
5. В **выберите проекты** диалогового окна поле, убедитесь, что **WingtipToys** выделения, а затем нажмите кнопку **ОК**. 

    ![Обработка ошибок ASP.NET - выберите проекты диалоговое окно](aspnet-error-handling/_static/image8.png)
6. Нажмите кнопку **закрыть** в **управление пакетами NuGet** диалоговое окно при необходимости.
7. Если Visual Studio запрашивает перезагрузить все открытые файлы, выберите «**Да для всех**».
8. Пакет ELMAH добавляет записи для себя в *Web.config* файл в корне проекта. Если Visual Studio предлагает вам требуется перезагрузить измененный *Web.config* файла, нажмите кнопку **Да**.

ELMAH готов для хранения необработанных возникающие ошибки.

### <a name="viewing-the-elmah-log"></a>Просмотр журнала ELMAH

Просмотр журнала ELMAH прост, но сначала вы создадите необрабатываемое исключение, который будет записываться в журнал ELMAH.

1. Нажмите клавишу **CTRL + F5** для запуска образца приложения Wingtip Toys.
2. Для записи в журнале ELMAH необработанное исключение, перейдите в браузере следующий URL-адрес (используя свой номер порта):  
    `https://localhost:44300/NoPage.aspx` Отображается страница «ошибка».
3. Чтобы отобразить журнал ELMAH, перейдите в браузере следующий URL-адрес (используя свой номер порта):  
    `https://localhost:44300/elmah.axd`
