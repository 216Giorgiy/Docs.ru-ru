## <a name="what-youll-learn"></a>Что вы узнаете следующее.

- Как изменить пользовательский Интерфейс для поддержки функций приложения внешнего хранилища Wingtip Toys.

Веб-форм ASP.NET позволяют создать динамическое содержимое для веб-приложения. Каждый веб-странице ASP.NET создается так же, как для статических HTML веб-страницы (страница не включает серверную обработку), но веб-страницы ASP.NET включают дополнительные элементы, которые ASP.NET распознает и обрабатывает для создания HTML-кода при запуске страницы.

Со статической страницы HTML (*.htm* или *.html* файл), сервер выполняет `Web` запроса путем считывания файла и отправкой как-в браузер. Напротив, когда кто-то запрашивает веб-страницу ASP.NET (*.aspx* файл), страница выполняется как программа на веб-сервере. Пока страница выполняется, он может выполнять любую задачу, требующую веб-сайта, включая вычисление значений, чтения или записи сведений о базе данных или вызов других программ. В качестве результата страница динамически создает разметку (таких как элементы в формате HTML) и отправляет этот динамический результат в браузере.

### <a name="page-directive"></a>@Page Директива

Веб-форм ASP.NET обычно содержат директивы, которые дают возможность указать страницу свойств и сведения о конфигурации для страницы. Директивы используются ASP.NET в качестве инструкции способ обработки страницы, но они не отображаются как часть разметки, который отправляется в браузер.

Наиболее часто используется директива `@Page` директивы, можно задать множество параметров конфигурации для страницы, включая следующие:

1. Серверный язык программирования для кода на странице, таких как C#.
2. Ли страница является страницей с использованием серверного кода непосредственно в страницу, которая называется страницы в одном файле или является ли страница с кодом в отдельном файле класса, который называется страница с выделенным кодом.
3. Ли страница имеет связанную главную страницу и поэтому должны рассматриваться как страницы содержимого.
4. Параметры отладки и трассировки.

Если не включать `@Page` директив на странице, или если эта директива не включает определенные настройки, параметр будет унаследован от *Web.config* файла конфигурации или из *Machine.config* файла конфигурации. *Machine.config* файл предоставляет дополнительные параметры конфигурации на все приложения, запущенные на компьютере.

> [!NOTE] 
> 
> *Machine.config* также предоставляет сведения обо всех параметрах возможные конфигурации.


### <a name="web-server-controls"></a>Элемент управления

В большинстве приложений веб-форм ASP.NET вы добавите элементы управления, которые позволяют пользователю взаимодействовать со страницей, таких как кнопки, текстовые поля, списки и т. д. Эти веб-сервера управления похожи на кнопки HTML и входных элементов. Тем не менее они обрабатываются на сервере, что позволяет использовать код сервера для задания их свойств. Эти элементы управления также вызывают события, которые можно обрабатывать в коде сервера.

Серверные элементы управления используют специальный синтаксис, который распознается ASP.NET при запуске страницы. Имена тегов для серверных элементов управления ASP.NET начинаются с `asp:` префикс. Это позволяет распознавать и обрабатывать эти серверные элементы управления ASP.NET. Префикс может отличаться, если элемент управления не является частью .NET Framework. В дополнение к `asp:` префикса, серверные элементы управления ASP.NET также включают `runat="server"` атрибута и `ID` , можно использовать для ссылки на элемент управления в серверном коде.

При запуске страницы ASP.NET определяет серверных элементов управления и выполняет код, который связан с этими элементами управления. Многие элементы управления отображают HTML или другие элементы разметки на странице, когда она появится в браузере.

### <a name="server-code"></a>Серверный код

Большинство приложений веб-форм ASP.NET включают код, выполняемый на сервере при обработке страницы. Как упоминалось выше, сервер код может использоваться для выполнения различных действий, таких как добавление данных в элементе управления ListView. ASP.NET поддерживает множество языков на сервере, включая C#, Visual Basic, J# и другим пользователям.

ASP.NET поддерживает две модели написания кода сервера для веб-страницы. В одном файле модели, код для страницы находится в элементе скрипта, в которых содержит открывающий тег `runat="server"` атрибута. Кроме того можно создать код для страницы в отдельный файл класса, которую называют модели кода. В этом случае страница веб-форм ASP.NET обычно не содержит сервер кода. Вместо этого `@Page` директива включает сведения, которые связывают *.aspx* страницу с ее файлом с выделенным кодом.

`CodeBehind` Атрибутов, содержащихся в `@Page` директива указывает имя отдельного файла класса и `Inherits` атрибут задает имя класса в файле кода, соответствующий странице.

Затем можно создать отдельные страницы содержимого, включающие содержимое, которое будет отображаться, как описано выше. Когда пользователи запрашивают страницы содержимого, ASP.NET объединяет их с главной страницей, чтобы получить выходные данные, соединяет макет главной страницы с содержимым на странице содержимого.

### <a name="master-pages"></a>Главные страницы

Главная страница представляет собой файл ASP.NET с расширением MASTER (например, *Site.Master*) с предопределенной структурой, который может содержать статический текст, HTML-элементов и серверных элементов управления. Главная страница идентифицируется специальный `@Master` директиву, которая заменяет `@Page` директивы, который используется для обычного *.aspx* страниц.

В дополнение к `@Master` директив, Главная страница также содержит все элементы HTML верхнего уровня для страницы, такие как `html`, `head`, и `form`. Например, на главной странице, добавленных выше, использовании HTML `table` макета, `img` элемент логотип компании, статический текст и элементы управления сервера для обработки распространенных членства для сайта. Любой код HTML и какие-либо элементы ASP.NET можно использовать как часть главную страницу.

Помимо статического текста и элементов управления, которые будут отображаться на всех страницах, Главная страница также содержит один или несколько **ContentPlaceHolder** элементов управления. Эти элементы управления местозаполнителя определяют области, где будет отображаться заменяемое содержимое. В свою очередь, заменяемое содержимое определяется на страницах содержимого, такие как *Default.aspx*, с использованием **содержимого** серверного элемента управления.

### <a name="running-the-application-and-creating-the-database"></a>Выполнение приложения и создание базы данных

Ранее в этой серии учебника вы инициализатор класса (с именем «ProductDatabaseInitializer») и с указанием этого класса в *global.asax.cs* файла. Платформа Entity Framework создает базу данных, когда приложение запускается первый раз, так как `Application_Start` метод, содержащийся в *global.asax.cs* файл будет вызывать класс инициализатора. Инициализатор класса будет использовать классы модели (`Category` и `Product`), добавленный ранее в этой серии учебника для создания базы данных.

1. В **обозревателе решений**, щелкните правой кнопкой мыши *Default.aspx* и выберите **задать в качестве начальной страницы**.
2. В Visual Studio нажмите клавишу **F5**.   
 Занимает немного времени, чтобы настроить все, что при первом запуске.   
    ![Пользовательский Интерфейс и навигации - окна браузера](ui_and_navigation/_static/image7.png)  
 При запуске приложения будет выполнена компиляция приложения и базы данных с именем *wingtiptoys.mdf* будет создан в *приложения\_данные* папки. В браузере появится меню навигации категории. Это меню был создан путем получения категорий из базы данных. В следующем уроке будет реализовывать навигации.
