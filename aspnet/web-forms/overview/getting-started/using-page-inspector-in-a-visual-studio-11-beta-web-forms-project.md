## <a name="enable-inspection-mode"></a>Включить режим инспектирования

Далее вы увидите, как работает режим инспектирования инспектора страниц. В окне инспектора страниц щелкните **инспектировать** кнопки.

![Проверки элемента](using-page-inspector-in-a-visual-studio-11-beta-web-forms-project/_static/image7.png)

Чтобы просмотреть режим инспектирования в действии, наведите курсор на различные части страницы в окне браузера инспектора страниц. После этого указатель мыши изменяется знака плюс и будет выделен под элементом:

![Наведении указателя мыши на div.content оболочки](using-page-inspector-in-a-visual-studio-11-beta-web-forms-project/_static/image8.png)

При перемещении указателя мыши, обратите внимание, что

- Содержимое в **источника** просмотра изменяется для отображения разметки, соответствующие выбранному элементу страницы. Соответствующие разметка выделяется цветом. Если источником является в другом файле, этот файл открывается в режиме исходного кода с выделяются соответствующую разметку.

- Разметка, отображаемая в **HTML** вкладку в инспекторе страниц также изменяется в соответствии с выбранного элемента на странице. В **HTML** вкладки, описанные соответствующую разметку.

- **Стили** вкладке отображаются правила CSS, относящиеся к текущему выбранному элементу.

<a id="_5_using_page"></a>

## <a name="use-page-inspector-to-make-changes-to-markup"></a>Для внесения изменений в разметке использовать инспектор страниц

Теперь вы увидите, как можно использовать для поиска и внести изменения в разметке или текст, расположение которого может оказаться неочевидным инспектора страниц.

Перевести инспектор страниц в режиме инспектирования, а затем прокрутите до нижней части домашней страницы.

Как только вводится области нижнего колонтитула, инспектор страниц будет открывать *Site.Master* файл макета в **источника** представление на временной вкладке справа от другой вкладки и выделяет раздел главного страницы, которые выбраны. Это показывает, как инспектор страниц можно найти и просмотреть содержимое на странице, которая фактически могут поступать из другого файла, чем тот, который был открыт изначально.

![Выделяет нижнего колонтитула в режиме инспектирования](using-page-inspector-in-a-visual-studio-11-beta-web-forms-project/_static/image9.png)

В окне браузера инспектора страниц наведите указатель мыши на строку с авторских прав <a id="a"> </a>Обратите внимание.

В *Site.Master* страницы, соответствующая строка будет выделена.

![Выделенной об авторских правах строке нижнего колонтитула](using-page-inspector-in-a-visual-studio-11-beta-web-forms-project/_static/image10.png)

Добавьте в конец строки в текст *Site.Master* файла.

&lt;p&gt;&amp;копирование; &lt;%: DateTime.Now.Year %&gt; -Rocks приложение My ASP.NET!&lt; /p&gt;

Сейчас нажмите клавиши Ctrl + Alt + Ввод или щелкните панель обновления, чтобы просмотреть результаты в окне браузера инспектора страниц.

![Мои Rocks приложения ASP.NET!](using-page-inspector-in-a-visual-studio-11-beta-web-forms-project/_static/image11.png)

Вы могли подумать, нижний колонтитул был на *Default.aspx* страницы, но принявшее значение master макет страницы и инспектор страниц найти его автоматически.

## <a name="css-auto-sync"></a>Автосинхронизация CSS

> [!NOTE]
> Этот компонент требуется версия 1.3 инспектора страниц.


Функция автоматической синхронизации CSS позволяет напрямую изменить CSS-файл и увидеть изменения непосредственно в браузере инспектора страниц.

**Syles** окне показаны все правила CSS для этого элемента. Прокрутите вниз до селектор класса поиска .featured .content оболочки. Щелкните «.featured .content-программу-оболочку». Инспектор страниц открывает CSS-файл, определяющий этот стиль (Site.css) и выделяет соответствующего стиля CSS.

![CSS-файл](using-page-inspector-in-a-visual-studio-11-beta-web-forms-project/_static/image18.png)

Теперь измените значение `background-color` для «red». Изменение немедленно отображается в браузере инспектора страниц.

### <a name="using-the-visual-studio-2012-css-color-picker"></a>С помощью Visual Studio 2012 палитра CSS

Редактор CSS в Visual Studio 2012 имеет палитру цветов, позволяющие легко выбрать и вставить цвета. Она имеет простой цветовую шкалу и выбора «pop вниз», которое обеспечивает более точное управление.

Палитра содержит стандартную палитру цветов и поддерживает стандартные имена цветов, хэш-кодов, цветов RGB, RGBA, HSL и HSLA ведет список цветов, которые вы недавно использовали в документе.

В строке, где свойство цвет фона было введите «bc» и нажмите клавишу со стрелкой вниз один раз.

При вводе первый символ каждого слова в свойства, разделенных дефисом, такого как «цвет фона» IntelliSense список будет отфильтрован и отображать только свойства, которые соответствуют:

![IntelliSense отфильтрованные значения](using-page-inspector-in-a-visual-studio-11-beta-web-forms-project/_static/image24.png)

Теперь введите двоеточие. При этом вставляется имя свойства полный цвет фона. Тип  **#**  или **rgb (**, и отображается палитра цветов:

![Палитра цветов CSS](using-page-inspector-in-a-visual-studio-11-beta-web-forms-project/_static/image25.png)

Чтобы увидеть, как работает средство выбора цветов, щелкните его цвета с указателем, или нажмите клавишу Стрелка вниз и затем с помощью клавиш со стрелками влево и вправо для обхода цвета. При посещении цвет, соответствующее значение для свойства цвет фона предварительного просмотра:

![значение свойства цвет фона предварительного просмотра](using-page-inspector-in-a-visual-studio-11-beta-web-forms-project/_static/image26.png)

На этом этапе можно нажать ВВОД, чтобы выбрать значения, а затем точку с запятой (;) для завершения операции CSS. Теперь перейдите к разделу, чтобы вы увидите, как работает средство выбора цвета pop раскрывающегося списка.

#### <a name="using-the-color-picker-pop-down"></a>С помощью средства выбора цвета Pop раскрывающегося списка

При цветовой шкалы не имеет точного цвет, который вы ищете, можно использовать палитру цветов pop раскрывающегося списка.

Чтобы открыть его, щелкните значок шеврона в правом конце цветовой шкалы или один или два раза нажмите клавишу Стрелка вниз на клавиатуре.

![Палитра цветов CSS Pop раскрывающегося списка](using-page-inspector-in-a-visual-studio-11-beta-web-forms-project/_static/image27.png)

Выберите цвет из вертикальная черта справа. Здесь показан градиент цвета в главном окне. Выберите цвет непосредственно на вертикальную панель, нажав клавишу ВВОД или щелкнув любую из точек в главном окне выберите с большей точностью.

Существует ли цвет на экране компьютера, который вы хотите использовать (это не обязательно должно находиться в пользовательском интерфейсе Visual Studio), его значение можно записать с помощью пипетки в правом нижнем углу.

Также можно изменить степени непрозрачности цвета, переместив ползунок в нижней части палитру цветов. Таким образом, изменения значения RGBA значения цветов, так как формат RGBA может представлять непрозрачности.

После выбора цвета, нажмите клавишу ВВОД, а затем введите точку с запятой для завершения операции цвет фона в *Site.css* файла.

### <a name="the-page-inspector-update-bar"></a>Панель обновления инспектора страницы

Инспектор страниц немедленно обнаруживает изменение *Site.css* файла (или любой другой файл в приложении) и отображает предупреждение на панели обновления.

![Панель обновления](using-page-inspector-in-a-visual-studio-11-beta-web-forms-project/_static/image28.png)

Чтобы сохранить все файлы и обновить страницу в браузере инспектора страниц, нажмите сочетание клавиш Ctrl + Alt + Ввод или щелкните панель обновления. Изменение цвета выделения откроется в браузере:

![Изменить цвет выделения](using-page-inspector-in-a-visual-studio-11-beta-web-forms-project/_static/image29.png)

<a id="_using_page_inspector_1"></a>Обратите внимание, удобным образом обновить браузер инспектора страниц прямо из среды Visual Studio. С помощью инспектора страниц вместо внешнем браузере позволяет оставаться в редакторе, при разработке веб-приложений.
